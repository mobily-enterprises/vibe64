      adapter contributes the target facts and commands.
    </p>
  </div>

  <div class="why-jskit-ai-ready-grid">
    <article class="why-jskit-ai-ready-card why-jskit-ai-ready-card--hero">
      <p class="why-jskit-card-label">A</p>
      <h2>Workflow steps are data</h2>
      <p>
        Steps define labels, descriptions, visible actions, enablement rules,
        rewind cleanup, and next-step requirements.
      </p>
    </article>
    <article class="why-jskit-ai-ready-card">
      <p class="why-jskit-card-label">B</p>
      <h2>Prompt actions get context</h2>
      <p>
        Prompt handoff includes the current session, action input, artifacts,
        project facts, adapter context, and workflow state.
      </p>
    </article>
    <article class="why-jskit-ai-ready-card">
      <p class="why-jskit-card-label">C</p>
      <h2>Commands are deterministic</h2>
      <p>
        Worktree creation, dependency installation, GitHub issue creation, and
        other deterministic work run as command-backed actions.
      </p>
    </article>
    <article class="why-jskit-ai-ready-card">
      <p class="why-jskit-card-label">D</p>
      <h2>Artifacts stay editable</h2>
      <p>
        Issue titles, issue bodies, PR drafts, and reviewable outputs are
        treated as files the human can inspect and revise.
      </p>
    </article>
    <article class="why-jskit-ai-ready-card">
      <p class="why-jskit-card-label">E</p>
      <h2>Rewind is honest</h2>
      <p>
        Rewind support clears step-specific action results, artifacts, and
        metadata so repeated review loops do not lie about state.
      </p>
    </article>
    <article class="why-jskit-ai-ready-card">
      <p class="why-jskit-card-label">F</p>
      <h2>Sessions live on disk</h2>
      <p>
        Runtime state is stored under <code>.vibe64</code>, with active
        session directories, metadata, artifacts, command logs, and step state.
      </p>
    </article>
