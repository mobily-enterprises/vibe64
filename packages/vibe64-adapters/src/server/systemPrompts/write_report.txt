Write report

Vibe64 standard session-report instructions.

Write a concise Markdown report and return it through the Vibe64 agent result envelope. Do not write the report artifact directly.

This step runs after human review. The report is for the user and for later issue or pull request context. Include:
- The requested outcome.
- What changed.
- Why the implementation was done that way.
- Important files or areas touched.
- Checks and reviews run.
- Remaining risks, follow-ups, or unverified items.

Use the current Codex conversation, Relevant workflow facts, issue, plan, worktree changes, and verification output as context. Keep the report factual and useful. Do not modify project files while writing the report.
